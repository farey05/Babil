3. Назовите виды компьютерной графики и их особенности.

Растровая графика:

Особенности: Представлена пикселями, обладает ограниченным разрешением. Подходит для фотографий и изображений с непрерывными цветами.
Примеры форматов: JPEG, PNG, GIF.
Векторная графика:

Особенности: Использует математические формулы для определения форм и линий, что позволяет масштабировать изображение без потери качества.
Примеры форматов: SVG, AI, EPS.
Точечная (битовая) графика:

Особенности: Использует отдельные точки для представления изображения. Часто применяется в мониторах с низким разрешением.
Примеры форматов: BMP, GIF.
Компьютерная анимация:

Особенности: Серия изображений, называемых кадрами, создает впечатление движения. Используется в мультфильмах, видеоиграх.
Примеры форматов: GIF, APNG, WebP.
Трехмерная графика:

Особенности: Использует трехмерные объекты для создания реалистичных изображений. Применяется в 3D-моделировании, играх и визуализации.
Примеры форматов: STL, OBJ, FBX.

1. Расскажите о технологиях верстки веб-приложений. Назовите разницу в стандартах языков разметки.

Технологии верстки веб-приложений:

HTML (HyperText Markup Language): Основной язык разметки, определяющий структуру контента веб-страницы.
CSS (Cascading Style Sheets): Язык стилей, применяемый для определения внешнего вида элементов, созданных с использованием HTML.
JavaScript: Язык программирования, который обеспечивает интерактивность и динамическое изменение содержимого веб-страницы.
CSS-препроцессоры (например, Sass, Less): Расширения CSS, добавляющие дополнительные функции, такие как переменные, вложенные правила и миксины.
Фреймворки (например, Bootstrap, Foundation): Наборы готовых компонентов и стилей для ускоренной разработки веб-приложений.
React, Vue, Angular: Библиотеки и фреймворки JavaScript для создания пользовательских интерфейсов и одностраничных приложений (SPA).
Разница в стандартах языков разметки:

HTML: Существуют различные версии, но основные стандарты включают HTML 4, XHTML (Extensible HyperText Markup Language), HTML5.
XML (eXtensible Markup Language): Общий стандарт, который может использоваться для создания различных языков разметки. Не связан напрямую с веб-страницами.

2. Дайте понятие fronted.Опишите все этапы разработки веб-сайта.

Определение: Frontend - это часть веб-разработки, отвечающая за создание пользовательского интерфейса и взаимодействие с пользователем.
Этапы разработки веб-сайта:
Планирование: Определение целей, аудитории, структуры сайта.
Дизайн: Создание макетов, выбор цветовой палитры, разработка дизайна.
Верстка (Frontend): Создание HTML, CSS, JavaScript кода для реализации дизайна и функциональности.
Backend: Разработка серверной части, базы данных, обработка бизнес-логики.
Тестирование: Проверка работоспособности, безопасности, совместимости.
Развёртывание: Размещение сайта на сервере, подключение домена.
Поддержка и обновление: Регулярное обслуживание, внесение изменений и обновлений.

3. Назовите основные понятия композиции.

Баланс: Равновесие между элементами композиции.
Контраст: Различие между элементами для привлечения внимания.
Повторение: Повторение определенных элементов для установления единого стиля.
Единство: Объединение всех элементов в цельную композицию.

4. Перечислите виды форм, дайте им характеристики.

Текстовая форма: Позволяет пользователю ввести текстовую информацию.
Выпадающий список: Предоставляет ограниченный набор вариантов для выбора.
Флажки (checkbox): Позволяют выбрать несколько вариантов из предложенного списка.
Переключатель (radio button): Позволяет выбрать один вариант из предложенного списка.
Текстовое поле: Предназначено для ввода числовых данных.
Кнопка отправки (submit): Инициирует отправку данных формы на сервер.
Кнопка сброса (reset): Сбрасывает введенные пользователем данные в форме.

1.  Дайте определение валидации. Какими способами можно создать валидацию данных формы?

Определение: Валидация - это процесс проверки данных, введенных пользователем в веб-форму, на соответствие установленным правилам и форматам.
Способы создания валидации данных формы:
HTML5 атрибуты: Использование атрибутов вроде required, pattern для указания обязательных полей и задания регулярных выражений.
JavaScript: Создание пользовательских функций в JavaScript для проверки данных, например, при отправке формы.
Бэкенд-валидация: Дополнительная проверка данных на сервере для обеспечения безопасности и целостности.
Библиотеки и фреймворки: Использование сторонних библиотек, таких как jQuery Validation Plugin или более современных фреймворков, для упрощения валидации.

2.  Расскажите про объектную модель документа и объектную модель браузера.

DOM: Объектная модель документа представляет собой иерархическую структуру объектов, представляющих элементы, атрибуты и текст на веб-странице. Она позволяет программам на языках, таких как JavaScript, взаимодействовать с содержимым веб-страницы.
Объектная модель браузера: Это набор объектов и методов, предоставляемых браузером для управления окном браузера, историей, загрузкой ресурсов и другими функциями. Она позволяет программам взаимодействовать с браузерным окружением.

3.  Как виды форм влияют на восприятие человека?

Текстовая форма: Позволяет вводить текстовую информацию. Влияет на восприятие, предоставляя удобный способ ввода текста.
Выпадающий список: Облегчает выбор из ограниченного списка вариантов, что может сделать форму более удобной.
Флажки (checkbox): Позволяют выбрать несколько вариантов, что полезно для множественного выбора. Влияют на восприятие, предоставляя гибкость.
Переключатель (radio button): Позволяет выбрать один вариант из нескольких. Управляет восприятием, когда требуется выбор только одного варианта.
Текстовое поле: Используется для ввода числовых данных. Влияет на восприятие, предоставляя удобный интерфейс для ввода чисел.

4.  Охарактеризуйте равновесие как композиционный приём.

Определение: Равновесие в композиции означает распределение визуального веса элементов на веб-странице так, чтобы создать гармоничное и уравновешенное восприятие.
Характеристики равновесия:
Симметричное равновесие: Элементы распределены симметрично относительно центральной оси, что придает структуру устойчивость.
Асимметричное равновесие: Элементы распределены несимметрично, но с учетом визуальных весов, что также создает гармонию.
Равномерное распределение: Элементы равномерно распределены по всей композиции, что предотвращает перегрузку одной области и поддерживает восприятие.
